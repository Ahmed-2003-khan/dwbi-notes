Intrinsic Data Quality: Data ki correctness / accuracy.

Realistic Data Quality: Data ka business ke liye value / usefulness.

3. Data Quality & Organizations

  • Intelligent Learning Organization: High-quality data = shared, open, value-adding resource.
  • Functional Learning Organization: Low-quality data = proprietary, cost-adding resource.

4. Orr’s Laws of Data Quality

  1. Data that is not used cannot be correct.
  2. Data quality is a function of its use, not collection.
  3. Data will be no better than its most stringent use.
  4. Data quality problems increase with system age.
  5. Rare events jab hotay hain to zyada traumatic hotay hain.

5. Total Quality Management (TQM)

  • Customer-oriented approach.
  • Includes product quality, process control, assurance, improvement.
  • Quality Assurance ≠ Quality Improvement.

6. Cost of Fixing Data Quality

Slide diagram: “Lowest Quality → Highest Quality” aur “Exponential Rise in Cost”.

Defect minimization economical hoti hai, defect elimination extremely expensive hoti hai.

Cost Types

  • Controllable Costs: Error analysis, correction.
  • Resultant Costs: Missed opportunities, external failures.
  • Equipment & Training Costs

8. Characteristics (Dimensions) of Data Quality

  • Accuracy: Lack of error
  • Completeness: Required values available
  • Believability: Data credible / trustworthy
  • Appropriate Amount of Data: Not too much, not too little
  • Timeliness: Up-to-date
  • Accessibility: Easily retrievable
  • Objectivity: Unbiased
  • Interpretability: Clear meaning, units

Completeness vs Accuracy

95% accurate + 100% complete V/S 100% accurate + 95% complete → depends on:

  • Tolerances
  • Application
  • Cost vs Benefit

9. Quantifying Data Quality

Ratios

  • Free-of-error ratio
  • Completeness ratio (entity, column, population)
  • Consistency ratio

Min-Max Technique

Multiple values aggregate karne ke liye. Min → conservative, Max → liberal.

Believability

Comparison with standard or expert knowledge.

10. Appropriate Amount of Data

Formula: Min {Dp/Dn , Dn/Dp}

Dp = Provided data units, Dn = Needed data units.

11. Timeliness & Accessibility

Timeliness

Timeliness = Max {0, 1 - C/V}
C = A + Dt - It

Accessibility

Accessibility = Max {0, 1 - Trd / Tru}
