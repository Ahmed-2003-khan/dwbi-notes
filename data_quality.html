1. Quality Kya Hai?

  • Formal: "Quality = requirements ko fulfill karna" (Crosby, 1979)
  • Webster: "Degree of excellence" — kisi cheez ka kitna acha hona

Auto Industry Example

Slide me diagram nahi tha, lekin concept yeh hai:
  • Quality ka matlab hai sirf customer ki needs meet karna, exceed karna zaruri nahi.
  • Focus un features pe jo customer ki life easy aur comfortable banate hain (safety, fuel efficiency, comfort).
  • Auto industry me improvements ka effect turant measure hota hai: customer satisfaction barhta hai, complaints kam hoti hain.
  • Lesson: Quality ka concept universal hai — chahe car ho ya data.

2. Data Kya Hai?

Data har real-world entity ka abstraction hota hai. Example:

Muhammad Khan
Height = 5'8"
Weight = 160 lbs
Gender = Male
Age = 35 yrs
Emp_ID = 440

Is se samajh aata hai ke data ka real-world reference hota hai, lekin wo hamesha digital / abstract form me hota hai.

3. Data Quality Kya Hai?

  • Intrinsic Data Quality: Data ki correctness ya accuracy
  • Realistic Data Quality: Data ka business ke liye utility ya usefulness

4. Data Quality aur Organizations

  • Intelligent Learning Organization: High-quality data = open, shared, value-adding resource
  • Functional Learning Organization: Low-quality data = proprietary, cost-adding resource

5. Orr’s Laws of Data Quality

  1. Unused data correct nahi ho sakta.
  2. Data quality use se define hoti hai, collection se nahi.
  3. Data quality hamesha sabse strict use ke barabar ya us se kam hoti hai.
  4. Purana system = zyada data quality problems.
  5. Rare events jab hotay hain, unka impact zyada hota hai.

6. Total Quality Management (TQM)

  • Sab log involve → systematic, continuous improvement
  • Customer-oriented approach
  • Include: Product quality, process control, quality assurance, improvement
  • Note: Quality Assurance ≠ Quality Improvement

7. Cost of Fixing Data Quality

Defect minimization economical hai, defect elimination bohot mehnga hai.

Lowest Quality -----------------> Highest Quality (Exponential rise in cost)

Cost Types:

  • Controllable Costs: Recurring cost for fixing data errors
  • Resultant Costs: Internal/external failures, missed opportunities
  • Equipment & Training Costs

8. Critical Areas for Data Quality

  • Marketing communication
  • Customer matching
  • Retail householding
  • Combining MIS systems after acquisition

9. Characteristics (Dimensions) of Data Quality

  • Accuracy: Data me error kam hona
  • Completeness: Required values available hon
  • Believability: Data trustworthy hona chahiye
  • Appropriate Amount of Data: Data quantity task ke liye sufficient
  • Timeliness: Data up-to-date
  • Accessibility: Easily retrievable
  • Objectivity: Unbiased
  • Interpretability: Clear meaning, units, symbols

Completeness vs Accuracy

Example: 95% accurate + 100% complete V/S 100% accurate + 95% complete → Decide based on tolerance, cost, business value, application.

10. Quantifying Data Quality

Metrics aur formulas:

  • Free-of-error ratio: 1 - (errors / total)
  • Completeness ratio: 1 - (incomplete items / total)
  • Consistency ratio: 1 - (violations / total checks)

Min-Max Technique

Multiple value aggregation ke liye. Min → conservative, Max → liberal

Believability Example

Min{0.8,0.7,0.6} = 0.6

Appropriate Amount of Data Formula

Min { Dp / Dn , Dn / Dp } 
Dp = Data Provided
Dn = Data Needed

Timeliness Formula

Max {0, 1 - C/V} 
C = Age + DeliveryTime - InputTime
V = Volatility (time data remains valid)

Accessibility Formula

Max {0, 1 - Trd / Tru} 
Trd = Request to delivery time
Tru = Useful duration of data

11. Data Quality Validation Techniques

  • Referential Integrity (RI)
  • Attribute Domain Validation
  • Data Quality Rules
  • Histogram / Statistical Validation

RI Example

Outstanding payments without matching customer_ID → DWH me weekly/monthly check hoti hai.

Attribute Domain Steps

  1. Count domain value occurrences
  2. Compare with valid set
  3. Investigate exceptions → cause + impact

Histogram Example

Spike of centurions (Age ≥100) → Outlier detection, normal environment me yeh rare hote hain

12. Data Quality Management Process (TDQM)

  1. Establish DQ Environment (Managers + Developers)
  2. Scope Projects & Develop Implementation Plans (Goals, Tasks, Milestones, Resources)
  3. Implement: Define → Measure → Analyze → Improve
  4. Evaluate Methods → Check improvement & modify if needed

13. Improving Data Quality

  • Process improvements
  • System upgrades
  • Policy & Procedure changes
  • Data Design fixes

14. Common Misconceptions

  • Data fix karna problem solve nahi karta — problem usage me hoti hai
  • Data Quality sirf IT ka kaam nahi — Business + IT joint effort
  • One-time tool purchase solution nahi hai

15. GIGO

Garbage In → Garbage Out → Agar input galat hoga to output bhi galat hoga
