DWBI Exam Notes (Dark Mode) Normalization, Denormalization & OLAP

1. Data Warehouse Types & Applications

A. Types of Data Warehouse

  • Financial DWH: Pehla DWH hota hai, kyunki financial data stable & important hota hai.
  • Telecom DWH: Bohat high-volume data (CDRs). Zyada data ko aggregate karke rakha jata hai.

B. Typical Applications

  • Fraud Detection: Patterns se fraud identify karna.
  • Profitability Analysis: Kaun sa customer profitable hai.
  • Targeted Marketing: Correct user ko correct offer.

2. Normalization

Definition: Redundancy hatana aur anomalies ko fix karna.

Goals

  • Data repetition remove karna.
  • Data anomalies ko avoid karna.

Anomalies

  • Insert Anomaly: Kuch data na ho to insert nahi hota.
  • Delete Anomaly: Ek record delete karo, important data bhi delete ho jaye.
  • Update Anomaly: Har jagah update na ho to mismatch.

Normalization Forms

  • 1NF: Atomic values only. Repeating groups nahi.
  • 2NF: No partial dependency (Composite key dependency issue solved).
  • 3NF: No transitive dependency.

3. Denormalization

Definition: Performance improve karne ke liye jaan-boojh kar duplication allow karna.
“It is not chaos — it is a controlled crash.”

Why Denormalization?

  • Joins kam ho jate hain → query fast.
  • DWH mein speed zyada important hoti hai.

5 Denormalization Techniques

# Technique Explanation (Roman Urdu)
1 Collapsing Tables 1:1 ya M:M tables ko merge kar dena.
2 Splitting Tables Horizontal (rows) & Vertical (columns) splitting.
3 Pre-Joining Frequently joined tables ko merge kar dena.
4 Adding Redundant Columns Join se bachne ke liye duplicate info add karna.
5 Derived Attributes Calculated values ko store kar lena (e.g., Age, Total Sales).

4. OLAP (Online Analytical Processing)

OLAP = Fast analytical processing for decision-making.

OLTP vs OLAP

Feature OLTP OLAP
Data LevelDetailedSummarized
OperationsInsert/Update/DeleteBulk Read
UserClerksManagers
Data AgeCurrentHistorical
SizeGBsTBs+

FASMI Test

  • F: Fast (5 seconds).
  • A: Analysis capable.
  • S: Shared & Secure.
  • M: Multidimensional.
  • I: Information access full.

MOLAP + Cube Operations

  • Drill Down: Year → Month.
  • Roll Up: Month → Year.
  • Slice & Dice: Particular segment select karna.
  • Pivot: View rotate karna.
Pros: Instant response. Cons: Heavy storage if sparse.
